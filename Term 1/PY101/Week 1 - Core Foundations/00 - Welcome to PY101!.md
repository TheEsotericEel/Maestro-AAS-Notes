# Welcome to PY101: Core Foundations

## 🎓 Course Overview
Welcome to **CS101: Introduction to Computer Science with Python**. 

This is not a "bootcamp" or a "crash course." This is a rigorous, foundational approach to software engineering. Our goal is not merely to teach you "how to code," but to teach you **how to think like a computer scientist**.

By the end of this 8-week term, you will move from zero knowledge to a level of **competence**, defined as the ability to solve novel problems by breaking them down into their component parts and constructing logical solutions using code.

### The Philosophy: First Principles
We adhere to a "First Principles" approach. We will not hand you magic spells or quick fixes. Instead, we will deconstruct every concept to its fundamental truth. You will understand **why** the computer behaves the way it does, not just **what** to type to make it work.

> [!QUOTE] The Zen of Python
> "Explicit is better than implicit."  
> "Simple is better than complex."  
> "Readability counts."  
> — *Tim Peters*

## 📅 Week 1: Speaking the Language
**Theme**: Vocabulary, Grammar, and Syntax

In this first week, we establish the bedrock of your programming knowledge. Just as you cannot write a novel without understanding sentence structure, you cannot build software without understanding the syntax of Python.

### Learning Objectives
| Objective | The "Why" | The "How" |
| :--- | :--- | :--- |
| **Output** | Programs are useless if they cannot communicate with the user. | `print()` function |
| **Memory** | Data must be stored and labeled to be manipulated later. | Variables and Assignment |
| **Data Types** | The computer must know *what* kind of data it is holding to know how to treat it. | `str`, `int`, `float`, `bool` |
| **Arithmetic** | At its core, a computer is a calculator. We must master its math logic. | Operators (`+`, `-`, `*`, `/`, `//`, `%`) |
| **Input** | Interaction requires a two-way street; we must receive data from the outside world. | `input()` function |
| **Debugging** | Failure is the default state. You must learn to read error messages to find the truth. | Traceback analysis |

## 🛠️ The Toolchain
A professional craftsman respects their tools. You will be using the industry-standard environment.

### 1. The Language: Python (3.10+)
We choose Python for its **readability** and **clarity**. It is the closest programming language to human English, allowing us to focus on *logic* rather than *syntax clutter*.
*   **Verification**: Open your terminal and type `python --version`. If it returns 3.10 or higher, you are ready.

### 2. The Editor: VS Code
We do not write code in Notepad. We use an **Integrated Development Environment (IDE)**. VS Code provides syntax highlighting, error detection, and debugging tools that act as an extension of your mind.

### 3. The Terminal
You will learn to command the computer directly via the Command Line Interface (CLI). This is the "control room" of your operating system.

## 🧠 The Programmer's Mindset
Success in this course is not determined by intelligence, but by **persistence**.

### 1. Computers are Literal
A computer is an evil genie. It will give you exactly what you asked for, not what you wanted.
*   **Example**: If you tell a robot to "go to the store and get milk," it might walk through a wall to get there. You didn't tell it to use the door.
*   **Lesson**: You must learn to be painfully precise.

### 2. Failure is Data
When your program crashes, it is not a personal failure. It is the computer communicating with you.
*   **Novice reaction**: "I'm not good at this."
*   **Engineer reaction**: "Interesting. The error says line 4. What is wrong with line 4?"

### 3. The Feedback Loop
Write code. Run code. Break code. Fix code.
This loop is how you learn. Do not be afraid to see red text. **Red text is the path to green text.**
